The flyer template schema (Phase 22 — verbatim primitives to copy):
```python
# flyer_generator/flyer/schema_renderer/schema_model.py
class FlyerTemplateSchema(BaseModel):
    schema_version: Literal["1"]
    name: str = Field(pattern=r"^[a-z][a-z0-9_]*$")
    description: str
    tone_keywords: list[str] = Field(default_factory=list)
    canvas: Canvas = Field(default_factory=Canvas)
    palette: Palette
    typography: Typography = Field(default_factory=Typography)
    panels: dict[_PanelName, PanelSchema]
```

Loader pattern (verbatim):
```python
# flyer_generator/flyer/schema_renderer/loader.py
_SCHEMAS_DIR = Path(__file__).parent.parent / "schemas"

def load_template(name_or_path: str) -> FlyerTemplateSchema:
    if name_or_path.endswith(".json"):
        path = Path(name_or_path)
    else:
        path = _SCHEMAS_DIR / f"{name_or_path}.json"
    if not path.exists():
        available = list_templates()
        raise FileNotFoundError(...)
    raw = json.loads(path.read_text(encoding="utf-8"))
    return FlyerTemplateSchema.model_validate(raw)
```

Differences for postcard:
- `_PanelName = Literal["front", "back"]` (not "hero", not the 6 brochure panels)
- Canvas has NO defaults — postcards have explicit dims: `Canvas(width: int, height: int)`
- Palette KEEPS scrim_opacity_top/bottom (front panel may use scrim over hero image)
- Validator REQUIRES both `front` AND `back` panel keys (mirror brochure's required-set check)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem → loader | `load_template(name_or_path)` resolves a JSON file from a user-supplied string. The bare-slug branch is constrained to `_SCHEMAS_DIR`, but the `.json`-suffix branch accepts ANY path. |
| JSON → Pydantic | Template JSON is parsed and validated; malformed input must fail loud (extra="forbid") not silently. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Tampering | `load_template` `.json`-suffix branch reads arbitrary path | accept (deferred to worker) | This plan exposes the loader; the worker plan (23-04) MUST add a `_validate_template_slug` guard rejecting `.json` suffix and `/` `\\` separators BEFORE the loader is called. Mirror Phase 22 T-22-10. |
| T-23-02 | Tampering | Malformed template JSON panics renderer late | mitigate | `extra="forbid"` on every BaseModel + required-panel validator forces failure at load time, before any rendering work. |
| T-23-03 | Information disclosure | FileNotFoundError leaks `_SCHEMAS_DIR` path | accept | The path is server-internal, the FE never displays raw exception strings. Same disposition as Phase 22 T-22-11. |
</threat_model>
