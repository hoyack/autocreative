**ComfyClient class** (per D-06 through D-12):

```python
import asyncio
import copy
from datetime import datetime, timezone
import structlog
import httpx
from flyer_generator.config import Settings
from flyer_generator.models import ComfyJob, GeneratedBackground
from flyer_generator.errors import (
    ComfySubmitError, ComfyJobFailedError, ComfyJobTimeoutError, ComfyDownloadError,
)

logger = structlog.get_logger()
```

1. **`__init__(self, settings: Settings, http_client: httpx.AsyncClient)`** (per D-06):
   - Store settings, http_client
   - `self._base_url = settings.comfycloud_base_url.rstrip("/")`
   - `self._headers = {"X-API-Key": settings.comfycloud_api_key.get_secret_value()}`

2. **`async def submit(self, workflow: ComfyWorkflow, attempt: int = 1) -> ComfyJob`** (per D-07, D-10):
   - POST to `{self._base_url}/api/prompt` with JSON body `{"prompt": workflow.workflow}` and headers `self._headers`
   - On 5xx: exponential backoff -- 3 retries starting at 1 second (1s, 2s, 4s). Implement as a loop with `asyncio.sleep(delay)` where `delay = 1.0 * (2 ** retry_attempt)`.
   - On 4xx or exhausted retries: raise `ComfySubmitError` with status code and response text
   - On success: extract `prompt_id` from response JSON
   - Return `ComfyJob(prompt_id=prompt_id, submitted_at=datetime.now(timezone.utc), positive_prompt=workflow.positive_prompt, negative_prompt=workflow.negative_prompt, seed=workflow.seed, attempt_number=attempt)`
   - Log: `logger.info("comfy_submitted", prompt_id=prompt_id)`

3. **`async def wait_for_completion(self, prompt_id: str) -> None`** (per D-08, D-09):
   - `await asyncio.sleep(self._settings.poll_initial_wait_seconds)` -- initial wait before first poll
   - Loop for `self._settings.poll_max_attempts` iterations:
     - GET `{self._base_url}/api/job/{prompt_id}/status` with headers
     - On 5xx: exponential backoff (same pattern as submit)
     - Extract `status = resp.json().get("status", "unknown")`
     - Log: `logger.info("comfy_poll", prompt_id=prompt_id, attempt=attempt, elapsed_s=round(elapsed, 1), status=status)`
     - If status in `("success", "completed")`: return (per D-09 + Pitfall 1 safety net for "completed")
     - If status in `("failed", "cancelled")`: raise `ComfyJobFailedError(f"ComfyCloud job {status}: {prompt_id}", prompt_id=prompt_id)`
     - Otherwise: `await asyncio.sleep(self._settings.poll_interval_seconds)`
   - After loop: raise `ComfyJobTimeoutError(f"ComfyCloud job timed out after {max_attempts} polls", prompt_id=prompt_id, attempts=max_attempts)`

4. **`async def download_result(self, prompt_id: str) -> bytes`** (per D-11):
   - GET `{self._base_url}/api/history_v2/{prompt_id}` with headers
   - On non-200: raise `ComfyDownloadError` with status and message
   - Parse response JSON to find the first image filename. The history_v2 response contains output info -- navigate the response to find the filename. Expected structure: look through response dict for output entries containing filenames with image extensions (.png, .jpg, .webp).
   - GET `{self._base_url}/api/view` with query param `filename={filename}` and headers
   - On non-200: raise `ComfyDownloadError`
   - Return response content (raw bytes)
   - Log: `logger.info("comfy_downloaded", prompt_id=prompt_id, size_bytes=len(content))`

5. **`async def generate(self, workflow: ComfyWorkflow, attempt: int = 1) -> tuple[ComfyJob, bytes]`** (per D-12):
   - Orchestrates: `job = await self.submit(workflow, attempt)` -> `await self.wait_for_completion(job.prompt_id)` -> `raw_bytes = await self.download_result(job.prompt_id)`
   - Return `(job, raw_bytes)` -- the caller (pipeline orchestrator in Phase 4) combines this with ImagePreprocessor to create GeneratedBackground
   - Log: `logger.info("comfy_generate_complete", prompt_id=job.prompt_id, attempt=attempt)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> ComfyCloud API | Untrusted network, API key in header, response data untrusted |
| ComfyCloud response -> application | Status values and image data from external service |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | S (Spoofing) | API key header | mitigate | API key stored as SecretStr in Settings, never logged (structlog won't expand SecretStr) |
| T-02-04 | T (Tampering) | status response | mitigate | Only accept known status values ("success", "completed", "failed", "cancelled"); unknown values trigger continued polling, not acceptance |
| T-02-05 | D (Denial of Service) | polling loop | mitigate | Max attempts cap (default 20), exponential backoff on 5xx (prevents hammering failed server) |
| T-02-06 | I (Info Disclosure) | error messages | mitigate | Do not include API key in error messages or logs; only include prompt_id and status |
| T-02-07 | T (Tampering) | downloaded image | accept | Image bytes passed to Pillow for upscale which validates PNG format; no additional validation needed at download |
</threat_model>
