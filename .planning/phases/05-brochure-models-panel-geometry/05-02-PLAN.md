<objective>
Implement the pure-function panel geometry for the tri-fold brochure. Given
the US Letter landscape format (11″×8.5″ at 300 DPI = 3300×2550 trim,
3375×2625 bleed canvas), compute six panel rectangles (3 outside, 3 inside),
their safe-zone insets (0.25″ = 75 px), fold-line x-coordinates, and
crop-mark corner points. Return the result as a strongly-typed
`ResolvedBrochureLayout` Pydantic model. No I/O, no logging, no external deps
beyond Pydantic.

Purpose: Phase 7 composer and Phase 8 PDF assembly both consume this layout
to place images, text, and crop marks with pixel-accurate alignment. Getting
the math right here — and locking it behind unit tests — means every later
phase inherits correct geometry for free.

Output:
- `flyer_generator/brochure/stages/layout.py` with constants, `PanelRect`, `ResolvedBrochureLayout`, and `compute_panel_layout() -> ResolvedBrochureLayout`
- `PanelName` Literal alias added to `flyer_generator/brochure/models.py` so both plans share the canonical panel-name type
- `flyer_generator/brochure/__init__.py` re-exports the new geometry symbols
- `tests/brochure/test_layout.py` enforcing the hard geometry invariants from D-05-19
</objective>

# flyer_generator/brochure/stages/layout.py (new)
from typing import Literal
from pydantic import BaseModel, ConfigDict, Field

from flyer_generator.brochure.models import PanelName

# D-05-12 — named constants; no magic numbers anywhere else
LETTER_LANDSCAPE_INCHES: tuple[float, float] = (11.0, 8.5)
BLEED_INCHES: float = 0.125
SAFE_INCHES: float = 0.25
DPI: int = 300

# Derived (kept module-level for consumers + tests)
TRIM_PX: tuple[int, int] = (3300, 2550)        # 11.0 * 300, 8.5 * 300
BLEED_CANVAS_PX: tuple[int, int] = (3375, 2625)  # trim + 2*0.125*300 each axis
BLEED_OFFSET_PX: int = 38  # int(round(0.125 * 300)) — shifts trim-origin inside bleed canvas; used in all panel math

SheetName = Literal["outside", "inside"]


class PanelRect(BaseModel):
    """Pixel rectangle for one of the six brochure panels."""

    model_config = ConfigDict(extra="forbid")

    name: PanelName
    index: int = Field(ge=1, le=6)       # D-05-15 — 1..6 (back_cover=6, front=1, tuck=2, inner_L/C/R = 3/4/5)
    sheet: SheetName
    bleed_rect: tuple[int, int, int, int]  # (x, y, w, h) on bleed canvas (3375x2625)
    trim_rect: tuple[int, int, int, int]   # (x, y, w, h) on bleed canvas; trim area only
    safe_rect: tuple[int, int, int, int]   # (x, y, w, h) on bleed canvas; safe zone inset by SAFE_INCHES


class ResolvedBrochureLayout(BaseModel):
    """Complete pre-computed layout for both tri-fold sheets."""

    model_config = ConfigDict(extra="forbid")

    outside_panels: list[PanelRect] = Field(min_length=3, max_length=3)
    inside_panels: list[PanelRect] = Field(min_length=3, max_length=3)
    fold_lines_outside: list[int] = Field(min_length=2, max_length=2)  # 2 x-coords on bleed canvas
    fold_lines_inside: list[int] = Field(min_length=2, max_length=2)
    crop_marks: list[tuple[int, int]] = Field(min_length=8, max_length=8)  # 8 corner points (2 sheets x 4)


def compute_panel_layout() -> ResolvedBrochureLayout:
    """Return the pixel-resolved layout for a US Letter landscape tri-fold brochure."""

    **Step C — Implement `compute_panel_layout()`:**

    Algorithm (pure function, no args):

    1. Precompute panel-within-trim geometry (same for every panel on both sheets):
       - Trim origin on the bleed canvas: `trim_x0 = BLEED_OFFSET_PX`, `trim_y0 = BLEED_OFFSET_PX`.
       - Three equal-width columns inside the trim area, each `PANEL_TRIM_W_PX` wide and `TRIM_H_PX` tall.
       - Column trim x-origins: `col0_x = trim_x0`, `col1_x = trim_x0 + PANEL_TRIM_W_PX`, `col2_x = trim_x0 + 2 * PANEL_TRIM_W_PX`.

    2. For the outside sheet (D-05-14: left→right = back_cover (idx 6), front_cover (idx 1), tuck_flap (idx 2)):
       - Column 0 → `back_cover`, index 6.
       - Column 1 → `front_cover`, index 1.
       - Column 2 → `tuck_flap`, index 2.

       For each, build trim_rect, then bleed_rect, then safe_rect:
       - **Leftmost column (col0)**: trim_rect = `(col0_x, trim_y0, PANEL_TRIM_W_PX, TRIM_H_PX)`; bleed_rect extends **left into the bleed margin**: `(0, 0, BLEED_OFFSET_PX + PANEL_TRIM_W_PX, BLEED_H_PX)` — covers the left bleed strip + top/bottom bleed strips for its column.
       - **Middle column (col1)**: trim_rect = `(col1_x, trim_y0, PANEL_TRIM_W_PX, TRIM_H_PX)`; bleed_rect extends **vertically only**: `(col1_x, 0, PANEL_TRIM_W_PX, BLEED_H_PX)`.
       - **Rightmost column (col2)**: trim_rect = `(col2_x, trim_y0, PANEL_TRIM_W_PX, TRIM_H_PX)`; bleed_rect extends **right into the bleed margin**: `(col2_x, 0, BLEED_OFFSET_PX + PANEL_TRIM_W_PX, BLEED_H_PX)`.
       - safe_rect for every panel: `(trim_x + SAFE_INSET_PX, trim_y + SAFE_INSET_PX, trim_w - 2*SAFE_INSET_PX, trim_h - 2*SAFE_INSET_PX)`.

       Invariant this satisfies: the three `bleed_rect`s tile the entire 3376×2626 bleed canvas with no overlap and no gap — left panel owns left bleed strip, right panel owns right bleed strip, each panel owns its full-height slice top→bottom.

    3. For the inside sheet (D-05-14: left→right = inner_left (idx 3), inner_center (idx 4), inner_right (idx 5)):
       - Column 0 → `inner_left`, index 3.
       - Column 1 → `inner_center`, index 4.
       - Column 2 → `inner_right`, index 5.
       - Geometry identical to the outside sheet per column; `sheet="inside"`.

    4. Build `fold_lines_outside = [trim_x0 + PANEL_TRIM_W_PX, trim_x0 + 2 * PANEL_TRIM_W_PX]` (two vertical fold x-coords on the bleed canvas). Same values for `fold_lines_inside`.

    5. Build `crop_marks` as 8 corner points (2 sheets × 4 corners, using the trim-corner positions on the bleed canvas — NOT inside the trim rect):
       ```python
       outside_corners = [
           (0, 0),                                           # top-left bleed corner (trim TL lies at (38,38), so (0,0) is in bleed)
           (BLEED_W_PX, 0),                                  # top-right
           (0, BLEED_H_PX),                                  # bottom-left
           (BLEED_W_PX, BLEED_H_PX),                         # bottom-right
       ]
       inside_corners = outside_corners  # same bleed canvas
       crop_marks = outside_corners + inside_corners
       ```
       Each point lies strictly outside the trim rectangle `[trim_x0, trim_y0, trim_x0+TRIM_W_PX, trim_y0+TRIM_H_PX]` — `(0,0)` has both x<38 and y<38, `(BLEED_W_PX, 0)` has x>=3338 and y<38, etc. This is what the test asserts.

    6. Return `ResolvedBrochureLayout(outside_panels=[...], inside_panels=[...], fold_lines_outside=[...], fold_lines_inside=[...], crop_marks=crop_marks)`.

    Keep the function pure: no logging, no I/O, no side effects, no mutable globals. Tests can call it N times and compare results for equality (D-05-18).
