<objective>
Refactor `FlyerGenerator.__init__` to accept an optional `canvas_dimensions: tuple[int, int] | None = None` kwarg. When None, fall back to (1080, 1920) — preserving byte-identical behavior for the existing flyer worker. When provided, the entire downstream pipeline (ImagePreprocessor upscale target, PosterComposer canvas math, Rasterizer output dims, FlyerOutput.dimensions) uses the injected size.

Purpose: PO-02 — establishes the pipeline reuse mechanism so the poster worker (24-04) can call `FlyerGenerator(canvas_dimensions=size_to_dim(size))` without forking the rendering code path. **CRITICAL:** the existing flyer worker calls `FlyerGenerator(settings=..., http_client=...)` with no canvas_dimensions kwarg — that call site MUST NOT change and MUST produce the same 1080×1920 output as before.

Output: A single non-breaking refactor across `pipeline.py`, `preprocessor.py`, `composer.py`, and `models.py`, with TDD-style tests proving back-compat (1080×1920 default still works) and new behavior (3 size presets produce correctly-sized PNGs).
</objective>

<patterns>
<!-- Critical patterns. -->

1. **Non-breaking default in constructor signature**: `canvas_dimensions: tuple[int, int] | None = None` with `or (1080, 1920)` fallback in __init__ body. The "or None then default" pattern is preferable to `tuple = (1080, 1920)` because it makes intent explicit and avoids the mutable-default trap (tuples are immutable but the pattern signals the kwarg is optional).

2. **Same kwarg pattern Rasterizer already uses** — `def __init__(self, width: int = 1080, height: int = 1920)`. Mirror this in PosterComposer and ImagePreprocessor.

3. **Composer's `_title_params` already takes anchor_x as a runtime arg** — easy to thread the dynamic canvas width through. The current code uses module-level `_CANVAS_WIDTH = 1080` and `_MARGIN_PX = 60`; convert these to instance attributes (`self._canvas_width`, `self._margin_px`).

4. **MARGIN_PX scales proportionally**: 60/1080 ≈ 5.56% of width. At 5400 wide, margin = 300; at 7200, margin = 400; at 8100, margin = 450. This keeps the visual relationship constant across size presets.

5. **Composer module-level constants `_CANVAS_WIDTH` + `_MARGIN_PX`** must be removed (or re-exported as derived defaults). Tests that reference them must adapt — but inspection confirms no test directly imports those constants by name.

6. **TDD RED → GREEN**: write failing tests in test_pipeline_canvas_dimensions.py, test_preprocessor_canvas_dimensions.py, and test_composer_canvas_dimensions.py FIRST. Run them, see failures (assertion mismatch on dims), then implement the refactor.
</patterns>
