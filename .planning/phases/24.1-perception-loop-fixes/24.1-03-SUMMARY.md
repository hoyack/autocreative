# Phase 24.1 Plan 03: Poster Vision Input Downsample Summary

PLF-04 fix: introduce `_downsample_for_vision` helper that caps the LLM image
payload at 1920 px on the long edge before base64 encoding, while preserving
the full-canvas `GeneratedBackground.image_bytes` for the composer / rasterizer
downstream.

## Bug behavior before fix

`POST /api/v1/posters` consistently failed with `httpx.WriteTimeout` raised
from inside `flyer_generator/stages/vision.py::_call_anthropic` →
`anthropic.AsyncAnthropic.messages.create`. Root cause: the upscaled poster
canvas (5400×7200 for 18×24, 7200×10800 for 24×36, 8100×12000 for 27×40)
encodes to ~25–50 MB of base64 in the request body. On a residential WSL2
upload the body did not finish writing within `FLYER_VISION_TIMEOUT_SECONDS`,
even at 180 s × 3 retries (~540 s wall-clock).

Failed jobs surfaced in the perception loop (`/tmp/perception/PERCEPTION-REPORT.md`,
Finding F7):

| arq job ID | provider | wall-clock | outcome |
|---|---|---|---|
| `01KQ28D29MH9E2NDDT3G5X7VHV` | ollama | 7 min | WriteTimeout |
| `01KQ29WJMBV957S183DYEA7EB4` | anthropic, 60s | 213 s | WriteTimeout |
| `01KQ2AM9EE6R3REJ25T67B26GN` | anthropic, 180s | 571 s | WriteTimeout |

Existing pytests did not catch this because `tests/poster/test_render_smoke.py`
mocks the vision call entirely and `tests/test_vision.py` builds 1080×1920
fixtures (which fit under any reasonable timeout).

## The fix

Single-helper change to `flyer_generator/stages/vision.py`:

```python
def _downsample_for_vision(
    image_bytes: bytes, *, max_long_edge: int = 1920
) -> bytes:
    """Resize to long_edge <= max_long_edge and re-encode as PNG."""
    try:
        with Image.open(io.BytesIO(image_bytes)) as img:
            img.load()
            w, h = img.size
            long_edge = max(w, h)
            if long_edge <= max_long_edge:
                return image_bytes
            scale = max_long_edge / long_edge
            new_size = (max(1, int(w * scale)), max(1, int(h * scale)))
            resized = img.resize(new_size, Image.LANCZOS)
            out = io.BytesIO()
            resized.convert("RGB").save(out, format="PNG", optimize=True)
            return out.getvalue()
    except (OSError, ValueError) as exc:
        logger.warning(
            "vision_downsample_skipped_unreadable_bytes",
            error=str(exc),
            byte_len=len(image_bytes),
        )
        return image_bytes
```

Integrated into `_evaluate_with_text` BEFORE the base64 encode:

```python
vision_bytes = _downsample_for_vision(image_bytes, max_long_edge=1920)
img_b64 = base64.b64encode(vision_bytes).decode()
```

`_call_anthropic` / `_call_anthropic_retry` / `_call_ollama` /
`_call_ollama_retry` are unchanged — they receive the already-downsampled
`img_b64` as a parameter, so retries don't re-encode and the
`media_type: "image/png"` field on the Anthropic content block stays correct.

`evaluate_cover` (brochure) also benefits since it routes through
`_evaluate_with_text`.

### Three properties locked by the helper

1. **No-op when long_edge ≤ 1920** — flyer / brochure 1080×1920 backgrounds
   pass through byte-identical, preserving any test that compares the bytes.
2. **PNG re-encode on resize** — the hardcoded `media_type: "image/png"` on
   both backends stays correct.
3. **Failure-safe on bad input** — PIL decode errors (e.g., `b"x"` sentinels
   in test mocks, or transient corruption in production) return the original
   bytes unchanged with a structured log warning. The downstream LLM already
   handles unreadable images via its existing rejection path.
