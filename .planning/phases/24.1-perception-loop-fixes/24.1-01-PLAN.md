<objective>
Fix PLF-01: postcard renders missing body text and never call Comfy. The
postcard worker (`flyer_generator/api/tasks/postcard.py`) must generate a hero
background from `image_hint` BEFORE invoking `render_postcard`, mirroring the
brochure worker's `generate_template_images` pattern. The two postcard schemas
(`classic_portrait.json`, `modern_landscape.json`) must place the `body` text
on the FRONT panel, not only on the back.

Purpose: A user POSTing `{headline, body, image_hint}` to `/api/v1/postcards`
expects the rendered front PNG to show the body copy AND a real image (not a
`[ hero ]` placeholder). Today the body never appears on the front and Comfy
is never called. Acceptance is in 24.1-CONTEXT.md ("PLF-01 — Postcard renderer
never composes body text or invokes Comfy").

Output:
- Updated postcard worker that calls Comfy when `image_hint` is supplied
- Updated `classic_portrait.json` + `modern_landscape.json` with a body
  TextElement on the front panel
- Updated `flyer_generator/postcard/schema_renderer/renderer.py` with an
  `images: dict[str, bytes] | None = None` kwarg on `render_postcard`
- Updated `tests/postcard/test_render_smoke.py` covering body-on-front in the
  rendered SVG (RED first, GREEN after schema fix)
- New worker test in `tests/api/test_worker_postcard_tasks.py` asserting
  Comfy is invoked **at the ComfyClient.generate seam** when `image_hint` is
  set (RED first, GREEN after worker fix)
</objective>
