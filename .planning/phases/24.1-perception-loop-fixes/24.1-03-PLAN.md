<objective>
Fix PLF-04: poster vision pipeline times out because the upscaled
5400×7200 (or larger) background PNG is sent in full to the LLM. Add a
`_downsample_for_vision` helper to `flyer_generator/stages/vision.py` that
resizes the image to a max long-edge of 1920 px and re-encodes as PNG, then
apply it inside `_evaluate_with_text` BEFORE the base64 encode. The original
`background.image_bytes` is preserved for the rest of the pipeline (composer
embeds it as the SVG background; rasterizer outputs at the full canvas).

Purpose: 5400×7200 PNGs encode to ~25–50 MB of base64 in the request body,
which times out on residential WSL2 uploads. A 1920-on-long-edge thumbnail is
~30× smaller and visually equivalent for zone-label classification (the LLM
picks 9-grid regions, not pixel-precise coordinates). This is purely an
internal optimization — no API contract change.

Output:
- `_downsample_for_vision(image_bytes, max_long_edge=1920) -> bytes` helper in
  `flyer_generator/stages/vision.py`
- Integration into `_evaluate_with_text` so both Anthropic and Ollama backends
  send downsampled bytes
- Unit test proving the helper caps the long edge at 1920 and re-encodes as
  valid PNG
- Integration test asserting the bytes passed into `_call_anthropic` (and
  `_call_ollama`) are downsampled when the input is large
- Regression: existing flyer/brochure/poster vision tests continue to pass

Scope clarification (from critical_decisions in 24.1-CONTEXT.md):
- "Don't touch the upstream pipeline — `background.image_bytes` retains full
  resolution for the rest of the stages." Confirmed: downsampling happens
  inside the vision stage, not in pipeline.py or preprocessor.py.
- The helper applies to BOTH `evaluate()` (poster + flyer event/info) AND
  `evaluate_cover()` (brochure cover). Posters are the urgent driver, but

Implementation skeleton (Pillow LANCZOS, no aspect-ratio change):
```python
import io
from PIL import Image

def _downsample_for_vision(image_bytes: bytes, *, max_long_edge: int = 1920) -> bytes:
    with Image.open(io.BytesIO(image_bytes)) as img:
        img.load()
        w, h = img.size
        long_edge = max(w, h)
        if long_edge <= max_long_edge:
            return image_bytes  # no-op for already-small inputs (preserves bytes)
        scale = max_long_edge / long_edge
        new_size = (int(w * scale), int(h * scale))
        resized = img.resize(new_size, Image.LANCZOS)
        out = io.BytesIO()
        # Match the existing image format ("image/png" is hardcoded in the
        # Anthropic + Ollama call sites, so always emit PNG to avoid a
        # media_type mismatch).
        resized.convert("RGB").save(out, format="PNG", optimize=True)
        return out.getvalue()
```

Why no-op when long_edge <= max: flyer/brochure backgrounds at 1080×1920 already fit. Skipping the encode/decode round-trip preserves byte-identical input on those paths and avoids breaking any existing test that compares bytes (none should, but defense-in-depth).

Pipeline integration (already in place — DO NOT change):


Evaluator already plumbs background.image_bytes into _evaluate_with_text via _call_backend (line 199). The downsample sits AFTER background.image_bytes is read into the local `image_bytes` parameter, so the upstream object is untouched.
