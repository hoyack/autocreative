<objective>
Fix PLF-03: the `bold_modern` flyer template's `cover_title` element has a
600-pixel-tall bbox starting at y=120 (which can extend down to y=720+ with
multi-line headlines and a 140px font), but the details elements (event.date,
event.time, event.location_name, event.fees) start at y=1300 with 60-pixel
bboxes. Per the perception loop screenshot
(`/tmp/perception/flyer-attempt2-render.png`), text-fit allows the headline
to render to a 4-5 line wrapped state that visually punches into the details
band. The fix is two-fold: (a) shrink the headline bbox height so it cannot
overflow into the details band, and (b) shift the details band slightly
upward into the freed space, with vertical separation. Add a deterministic
pytest that asserts no headline-vs-details overlap for ALL 6 flyer templates.

Purpose: A user submitting `template="bold_modern"` with a long event title
("SPRING TECH MEETUP") sees the title text and the date/time block render in
the same vertical band — letters intersect. The acceptance bullet from
CONTEXT.md: "Re-render with the same input on `bold_modern` produces no
visual overlap between headline and details."

Output:
- One small JSON edit to `flyer_generator/flyer/schemas/bold_modern.json`
  reshaping the cover_title bbox + nudging the details block
- One new deterministic test file `tests/flyer/schema_renderer/test_template_layouts.py`
  that loads each of the 6 templates and asserts no overlapping bboxes
  between cover_title and event.* (or any body-role) elements

Scope clarification (from critical_decisions in 24.1-CONTEXT.md):
- "Likely a template-data fix, not a renderer-code fix." Confirmed: we touch
  ONE JSON file. No code changes to PosterComposer or the schema_renderer.
- "Compare side-by-side with editorial_classic.json (which works) and shift
  the details block down enough to clear the headline's bottom edge." We do
  the inverse — shrink the title bbox AND shift details up — because
  bold_modern's title is at the TOP (y=120) and details are at the BOTTOM
  (y=1300+), unlike editorial_classic where details sit just below the title
  near y=550-700. The collision in bold_modern is caused by the title's
  600px height allowing it to wrap into the details band. Constraining the
  title's height to ~four lines at 140px (~700px including line-height) is
  the right answer.
</objective>

The current bold_modern.json bbox layout (the bug):
```json
"hero": {
  "elements": [
    { "type": "image_placeholder", "bbox": [0, 0, 1080, 1920], "slot": "hero", "z": 0 },
    { "type": "shape", "rect": [0, 1800, 1080, 120], "z": 5 },
    { "type": "text", "bbox": [60, 120, 960, 600], "role": "cover_title",
      "content_key": "event.title", "uppercase": true, "valign": "top", "z": 10 },
    // ↑ title can render down to y=720+ (start) + N lines × 147 line-height
    { "type": "text", "bbox": [60, 1300, 960, 60], "role": "body",
      "content_key": "event.date", "z": 10 },
    { "type": "text", "bbox": [60, 1370, 960, 60], "role": "body",
      "content_key": "event.time", "z": 10 },
    { "type": "text", "bbox": [60, 1440, 960, 60], "role": "body",
      "content_key": "event.location_name", "z": 10 },
    { "type": "text", "bbox": [60, 1510, 960, 60], "role": "body",
      "content_key": "event.fees", "z": 10 },
    { "type": "text", "bbox": [60, 1830, 960, 60], "role": "org_name",
      "content_key": "org", "z": 20 }
  ]
}
```

Canvas is 1080×1920. cover_title_size=140, body_size=38.

Element bbox semantics (from `_render_text_element` in `flyer_generator/brochure/schema_renderer/renderer.py`, mirrored in flyer renderer):
- bbox is `[x, y, w, h]` where (x,y) is top-left, w is width, h is height
- `valign: "top"` makes y_start = y + size; subsequent lines step by line_height
- The renderer does NOT clip text to bbox height — `fit_to_bbox` calls
  `chars_per_line` for width, but the line count is whatever the wrapped
  text needs. So a 600px-tall bbox at 140px font could hold ~4 lines (147
  line-height each), but a poorly-wrapped 5-word title could overflow.

The fix proposal (this plan executes this exact set of bbox edits):
```json
"hero": {
  "elements": [
    { "type": "image_placeholder", "bbox": [0, 0, 1080, 1920], "slot": "hero", "z": 0 },
    { "type": "shape", "rect": [0, 1800, 1080, 120], "z": 5 },
    // SHRINK cover_title height 600 → 480 (max ~3 lines at 147 line-height)
    { "type": "text", "bbox": [60, 120, 960, 480], "role": "cover_title",
      "content_key": "event.title", "uppercase": true, "valign": "top", "z": 10 },
    // SHIFT details band UP by 100px (1300 → 1200) to leave 240px gap
    // from headline bottom (120 + 480 = 600) to details top (1200) — wide enough
    // to defeat any overflow plus add visual breathing room.
    { "type": "text", "bbox": [60, 1200, 960, 60], "role": "body", "content_key": "event.date", "z": 10 },
    { "type": "text", "bbox": [60, 1270, 960, 60], "role": "body", "content_key": "event.time", "z": 10 },
    { "type": "text", "bbox": [60, 1340, 960, 60], "role": "body", "content_key": "event.location_name", "z": 10 },
    { "type": "text", "bbox": [60, 1410, 960, 60], "role": "body", "content_key": "event.fees", "z": 10 },
    // org_name stays at y=1830 inside the bottom accent stripe — unchanged
    { "type": "text", "bbox": [60, 1830, 960, 60], "role": "org_name", "content_key": "org", "z": 20 }
  ]
}
```

Result: title spans y=120..600 (max), details spans y=1200..1470. Vertical
gap = 600px, an unambiguous separation that a 4-line headline cannot bridge.

                 ax + aw <= bx  # a is left of b
