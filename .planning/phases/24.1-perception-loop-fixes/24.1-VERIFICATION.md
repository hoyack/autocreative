### Human Verification Required

#### 1. Poster 18x24 residential end-to-end timing

**Test:** Start the arq worker with `FLYER_VISION_TIMEOUT_SECONDS=60`, POST to `/api/v1/posters` with `size="18x24"`, `template="editorial_grand"` (or any template), realistic event data, and `image_hint` set. Poll until job status is `succeeded` or `failed`.

**Expected:** Job status transitions to `succeeded` with a rendered PNG artifact. Total wall-clock time from POST to succeeded status is under 120 seconds. No `httpx.WriteTimeout` in the worker log.

**Why human:** Anthropic API upload latency depends on the client machine's upload bandwidth. The 5400x7200 PNG was encoding to ~25-50MB before the fix; after the fix the vision input is ~1440x1920 (~1-2MB). Whether 1-2MB uploads within 60s on a specific residential WSL2 connection cannot be asserted in CI.

#### 2. All 3 poster sizes within FLYER_VISION_TIMEOUT_SECONDS=60

**Test:** Run three sequential (or parallel) poster jobs with `size` set to each of `"18x24"`, `"24x36"`, and `"27x40"`. Confirm all 3 reach `succeeded` status with `FLYER_VISION_TIMEOUT_SECONDS=60`.

**Expected:** All 3 jobs succeed. Worker logs show `vision_approved` or `vision_rejected` (not `WriteTimeout`). The larger 27x40 poster (canvas 8100x12000) downsamples to ~1080x1920 before vision — the base64 payload is the same ~1-2MB as for 18x24.

**Why human:** Requires real ComfyCloud round-trips (Comfy image generation) + real Anthropic vision calls against actual residential network conditions. Even if the code is correct, a flaky Comfy job or an Anthropic service disruption could produce a false failure. This is a performance UAT gate, not a correctness gate.
