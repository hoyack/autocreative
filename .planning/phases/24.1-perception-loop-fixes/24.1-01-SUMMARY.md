# Phase 24.1 Plan 01: PLF-01 Postcard Renderer Fix — Comfy + Body Text Summary

**Postcard front PNG now renders body text and hydrates a real Comfy hero image when image_hint is supplied — eliminates the perception-loop "[ hero ]" placeholder failure end-to-end at the lowest integration seam.**

## Performance

- **Duration:** ~25 min (TDD RED → GREEN → regression sweep)
- **Tasks:** 3 (RED tests, GREEN implementation, regression sweep + summary)
- **Files modified:** 6 (4 production, 2 test) + 1 created (`image_gate.py`)
- **Commits:** 2 atomic + this SUMMARY commit

## Accomplishments

- **Body copy now visible on the postcard FRONT panel** for both shipped templates (`classic_portrait` and `modern_landscape`). Before the fix, only the headline rendered on the front; the rest of the message lived on the back and was invisible until the user flipped the card. Perception loop renders that lacked the body string `Garden Tour starts at 11am` will now contain it.
- **Postcard worker invokes Comfy when `image_hint` is supplied**, mirroring the brochure pipeline pattern. The new `generate_postcard_hero` helper routes through the canonical `generate_single_image` seam (same one used by `social/generator.py`), so patching `flyer_generator.stages.comfy_client.ComfyClient.generate` directly intercepts the outbound call — proving the real wiring rather than a mock-friendly seam.
- **Literal `[ hero ]` placeholder text gone from rendered front PNGs.** Both shipped schemas now have `show_placeholder_label: false` so even when ComfyCloud is unreachable and the renderer falls back to its gradient placeholder, no debug-style label leaks into a user-facing render.
- **Renderer image-hydration parity with brochure.** `render_postcard` accepts an `images: dict[str, bytes] | None` kwarg and uses a brochure-style `_embed_image` helper to base64-embed the PNG into an `<image>` tag with optional clipPath for circle/rounded masks. Default `None` keeps every existing positional caller and test intact.

## Bug → Fix Traceability

**Bug behavior before fix** (perception loop run, 2026-04-25):
- `/tmp/perception/postcard-attempt1-render.png` (`classic_portrait`, 514 KB): "Save the Date" + literal `[ hero ]`, no body.
- `/tmp/perception/postcard-attempt2-render.png` (`modern_landscape`, 273 KB): same shape, different layout.
- Worker log: no `comfy_submitted` line for postcard tasks (job ran in 3.0 s — no Comfy call).
- Adversarial vision verdict: 3/10, "complete production failure".

**Two-pronged fix:**
1. **Schema fix** — body element on FRONT for both templates + `show_placeholder_label: false`. The body is the message; users expect it to be readable without flipping the card.
2. **Worker fix** — Comfy call wired in mirroring brochure (`generate_postcard_hero` → `generate_single_image` → `ComfyClient.generate`). Test patches the lowest seam so the contract is locked at the real outbound wiring.

## Decisions Made

- **Hero gen via `generate_single_image` seam, not a postcard-specific Comfy wrapper.** Same pattern as `social/generator.py` and `social/campaign.py`; keeps the integration surface narrow (one canonical seam to mock).
- **No vision gate on postcard hero.** CONTEXT.md locked the bug as "no Comfy call is made"; adding a vision gate would be scope creep into Phase 26 (adversarial hardening).
- **Failures swallowed with structured warning log.** Mirrors the brochure spot-image policy — postcards must always render, the gradient placeholder is a safety net.
- **Images passed as kwarg, not positional, into `render_postcard`.** Lets the worker's `render_postcard(template, content, images=images)` call coexist with existing 2-arg fake_render stubs that only need a tiny `images=None` extension to be forward-compatible.
- **Headline bbox shrink in `classic_portrait`.** Reducing height 300 → 220 was necessary to make the body element fit inside the bottom scrim. Verified the existing 2-line XML-escape headline test still passes.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Existing fake_render closures broke after kwarg addition**
- **Found during:** Task 2 (GREEN — wiring `images` into `render_postcard`)
- **Issue:** Two existing tests (`test_task_generate_postcard_threads_address_block`, `test_task_generate_postcard_no_address`) defined `def fake_render(template, content)` as `side_effect=fake_render`. Passing `images=images` as a kwarg from the worker raised `TypeError: got an unexpected keyword argument 'images'`.
- **Fix:** Extended both fake_render closures to accept `images=None`. The renderer's contract evolved (kwarg added with default), and the test stubs need to track that contract.
- **Files modified:** `tests/api/test_worker_postcard_tasks.py`
- **Verification:** All 16 worker tests now pass.
- **Committed in:** `4bb7135` (Task 2 commit)

## Issues Encountered

- **Initial classic_portrait headline bbox shrunk too aggressively.** First attempt set headline bbox height to 110 px to fit body in 270 px below it. The existing `test_front_svg_xml_escapes_headline` test passes a 23-char headline that wraps to 2 lines under `fit_to_bbox`; with height 110 px the second line was dropped, breaking `&lt;World&gt;` containment. Fixed by re-balancing to headline=[80, 1340, 1040, 220] + body=[80, 1580, 1040, 190], giving the headline ≥2 lines while still leaving the body 190 px in the scrim region.
