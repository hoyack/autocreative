# Phase 24.1 Plan 02: Brochure PLF-02 Fix Summary

**One-liner:** Replaced editorial_classic.json's hardcoded `static_text: "ESTATE PLANNING"` kicker with `content_key: "tagline"` and flipped `show_placeholder_label: false` on every brochure image_placeholder, eliminating both PLF-02 acceptance failures (#3 hardcoded kicker, #4 literal `[ hero ]` text).

## Bug Behavior Before Fix

The perception loop's adversarial run (2026-04-25) on `POST /api/v1/brochures` with:

```json
{
  "template": "editorial_classic",
  "content": {
    "title": "...",
    "tagline": "Spring Update",
    "sections": [
      {"heading": "Our Vision", ...},
      {"heading": "By the Numbers", ...},
      {"heading": "Looking Forward", ...}
    ],
    "hero_concept": "lush spring garden"
  },
  "generate_images": true
}
```

produced (`/tmp/perception/brochure-attempt1-render.png`):
1. Front-cover kicker rendered as `"ESTATE PLANNING"` — that string was nowhere in the brief; it was the literal value of `static_text` baked into the schema JSON.
2. Hero panel showed the literal `"[ hero ]"` placeholder label, not a Comfy-generated image.
3. The perception report flagged sections as missing (later confirmed in Task 1 investigation to be either a vision misread or brief-shape issue — the rendering path itself is correct).

## Investigation Findings (Task 1)

Documented inline in `tests/brochure/schema_renderer/test_renderer.py` top-of-module comment.

| Hypothesis | Evidence | Verdict |
|---|---|---|
| `tasks/brochure.py` drops content.sections | Worker calls `BrochureContent.model_validate(payload["content"])` (line 57) and passes the model unchanged to `render_schema_brochure` (lines 89-95) | INTACT |
| `renderer.py` doesn't iterate sections | `_resolve_text_content` calls `content.resolve_key(el.content_key, el.section_index)`; `BrochureContent.resolve_key` parses `"sections[N].field"` correctly | INTACT |
| `editorial_classic.json` lacks section content_keys | inner_left/center/right reference `sections[0..2].heading|lead_paragraph|bullets|quote`, all with explicit `section_index` | PRESENT |

**Conclusion:** Path A (JSON-only fix) is sufficient. The PLF-02 root cause is the hardcoded `static_text: "ESTATE PLANNING"` on `editorial_classic.json` line 33. The new test `test_editorial_classic_renders_all_supplied_sections` verifies the sections-rendering path works correctly today (it passed before any code change, locking the contract going forward).

## The Micro-Fixes

### 1. Kicker → content.tagline (editorial_classic.json line 33)

**Before:**
```json
{ "type": "text", "bbox": [80, 1580, 940, 60], "role": "static",
  "static_text": "ESTATE PLANNING",
  "font_size": 26, ... }
```

**After:**
```json
{ "type": "text", "bbox": [80, 1580, 940, 60], "role": "tagline",
  "content_key": "tagline",
  "font_size": 26, ... }
```

`_resolve_text_content` now reads `content.tagline`. `uppercase: true` is preserved so a tagline of "Spring Update" renders as "SPRING UPDATE". When tagline is `None`, the empty-text branch in `_render_text_element` skips the element — no fallback to the legacy hardcoded string.

### 2. Eliminate `[ slot ]` placeholder labels (all 13 brochure schemas)

The `ImagePlaceholder` schema model defaults `show_placeholder_label: bool = True` (line 266 of `schema_model.py`), which renders the literal text `[ hero ]` / `[ spot_1 ]` over the gradient fallback when Comfy doesn't provide an image. Per PLF-02 acceptance #4, that text must NOT appear in any rendered brochure.

Audited every `image_placeholder` element in `flyer_generator/brochure/schemas/*.json` and added `"show_placeholder_label": false` to each (24 elements across 11 schemas were missing the explicit `false`; only `editorial_classic.json` had the explicit `true` that was flipped). Final state: every brochure image_placeholder is now explicitly `false`. Format-preserving edits — only the new field was inserted, no other reformatting.

### 3. Worker-layer real-bug-surface test

`tests/api/test_worker_brochure_tasks.py` (new file) patches `ComfyClient.generate` directly via `patch.object(ComfyClient, "generate", new=AsyncMock(...))` and asserts:

- When `generate_images: true` + `content.hero_concept` is set, `ComfyClient.generate` is awaited at least once (the worker actually wires Comfy at runtime, not just `image_gate` in isolation).
- When `generate_images: false`, `ComfyClient.generate` is never awaited (the inverse contract).
