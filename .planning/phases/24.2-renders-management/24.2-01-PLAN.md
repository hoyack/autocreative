<objective>
Fix the print PDF page-size bug in BOTH brochure and postcard PDF assemblers. ReportLab interprets the `pagesize=(w, h)` tuple as PostScript points (1pt = 1/72in), but both modules currently pass 300dpi pixel values directly — yielding a brochure mediabox of 3376x2626pt = 46.89×36.47in instead of the correct 810x630pt = 11.25×8.75in (a 4.17× over-size factor of 300/72), and analogous oversize for postcards (1200x1800pt = 16.67×25in instead of 4×6in).

Fix per CONTEXT.md decision: declare a module-level constant `PT_PER_PX = 72.0 / 300.0` in each `pdf.py`, pass `pagesize=(w_px * PT_PER_PX, h_px * PT_PER_PX)` to `Canvas(...)`, then `canvas.scale(PT_PER_PX, PT_PER_PX)` so user-space stays in 300dpi pixel units. `drawImage(..., width=page_w_px, height=page_h_px)` and `_draw_crop_marks` (brochure only) keep their existing pixel-coordinate values unchanged. Two-line surgical fix in each file; no refactor of `_draw_crop_marks`, no refactor of `flyer_generator/brochure/stages/layout.py` constants.

Purpose: RM-01. Brochure + postcard PDFs are currently unprintable on commercial press hardware because the page boxes exceed the trim guides commercial printers expect. After this plan, `pypdf.PdfReader(...).pages[0].mediabox.width / 72` equals `11.25` for brochures and `4.0`/`6.0` for postcards depending on orientation.

Output: 2 production-code edits (~3 LOC change each + 1 const), 1 new pytest file under `tests/brochure/stages/test_pdf_dimensions.py` (with new `tests/brochure/stages/__init__.py` package marker), 1 new pytest file under `tests/postcard/stages/test_pdf_dimensions.py`, 1 small update to `tests/brochure/test_pdf.py::test_pdf_page_size_matches_bleed_canvas` (it currently asserts the bug as the contract — flip to assert points-based mediabox).
</objective>

<!-- The 72/300 conversion pattern. Apply to both pdf.py files. The user-space
     scale call ensures every other reportlab call (drawImage, line, setLineWidth)
     keeps using pixel coordinates so no other code in the module needs to change. -->

```python
# Module-level constant (each pdf.py)
PT_PER_PX: float = 72.0 / 300.0  # 0.24 — ReportLab points per 300dpi pixel

# In assemble_*_pdf:
page_w_pt = page_width_px * PT_PER_PX
page_h_pt = page_height_px * PT_PER_PX
canvas = Canvas(buf, pagesize=(page_w_pt, page_h_pt))
canvas.scale(PT_PER_PX, PT_PER_PX)  # user-space stays in 300dpi pixels
canvas.drawImage(
    ImageReader(io.BytesIO(png_bytes)),
    0, 0,
    width=page_width_px, height=page_height_px,
    preserveAspectRatio=False,
)
# Brochure crop marks already use pixel coordinates — no change.
canvas.showPage()
```
