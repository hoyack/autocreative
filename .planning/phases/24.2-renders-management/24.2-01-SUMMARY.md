## Decisions Made

- **`canvas.scale(...)` over coordinate translation in callers.** Calling `canvas.scale(PT_PER_PX, PT_PER_PX)` means every other reportlab call (`drawImage`, `setLineWidth`, `line`, `_draw_crop_marks`) keeps using pixel coordinates. Zero cascading edits to `layout.py` constants, `_draw_crop_marks` line lengths, or any caller — the fix is truly surgical (~3 LOC + 1 const each file).
- **Re-apply scale after `showPage()`.** ReportLab resets the CTM (current transformation matrix) at page break. Without re-applying, page 2's drawImage would render at 0.24x of intended size. Verified by the Page-2 dimension tests + existing color-distinguished page-2 image-XObject smoke tests passing.
- **Local module-level constant per pdf.py, not a shared module.** Per CONTEXT.md decision and PLAN frontmatter `key_links` pattern (`PT_PER_PX|72.0 / 300.0`), each pdf.py owns its own constant — local, simple, no abstraction debt.
- **Flip existing tests rather than delete.** The pre-existing `test_pdf_page_size_matches_bleed_canvas` (brochure), `test_portrait_page_*_dimensions_match_input` + `test_landscape_dimensions_round_trip` (postcard stages), and `test_render_smoke_all_permutations[*]` mediabox checks (postcard render smoke) had codified the bug as the contract. They remain the dimensional contract guard — the contract just changes from "buggy pixels" to "correct points."
