<objective>
Close two review warnings in `flyer_generator/api/routes/brand_kits.py`:

**WR-02**: `list_brand_kits` (lines 121-173) builds `db_slugs` from only the current page's DB rows, then the filesystem fuse adds any FS slug not in that set. When a slug exists in the DB but on a *different* page, it will be incorrectly treated as FS-only on the current page — inflating `total` and producing a duplicate row across pages. Also, `total = db_total + fs_only_count` uses `fs_only_count` from a single page's iteration of the FS, so the reported total is unstable.

**WR-03 (brand-kits half)**: `create_brand_kit_fetch` (lines 54-84) commits the JobRecord before calling `arq_pool.enqueue_job`. If enqueue raises, the row sits in `QUEUED` forever with no worker to advance it. Same pattern as the brochure enqueue failure (addressed in Plan 21-12 Task 2).

Purpose: Fix two data-integrity bugs in the brand-kit route — stable pagination counts + no orphan QUEUED rows on enqueue failure.

Output: Rewritten `list_brand_kits` with full-DB-slug dedup + stable total; try/except wrap in `create_brand_kit_fetch`; two regression tests.

    ```python
    @router.get(
        "/brand-kits",
        response_model=PaginatedBrandKits,
        summary="List brand kits (DB + filesystem fuse)",
    )
    async def list_brand_kits(
        limit: int = Query(default=50, ge=1, le=200),
        offset: int = Query(default=0, ge=0),
        session: AsyncSession = Depends(get_session),
        settings: AppSettings = Depends(get_settings),
    ) -> PaginatedBrandKits:
        """Return DB rows unioned with filesystem-only kits.

        WR-02 fix: the dedup key set is the FULL set of DB slugs (cheap single
        indexed-column SELECT), not just the current page's slice. This makes
        the `total` count page-invariant and prevents a slug that appears in
        the DB on page N from being misreported as "FS-only" on page N-1.
        """
        # 1. Full DB slug set for dedup + full DB count (both cheap).
        db_total = (
            await session.execute(select(func.count()).select_from(BrandKitRecord))
        ).scalar_one()
        all_db_slugs: set[str] = set(
            (await session.execute(select(BrandKitRecord.slug))).scalars().all()
        )

        # 2. Enumerate FS-only slugs ONCE (page-invariant). The v1 dataset is
        #    expected to be a few dozen kits at most (single-user private
        #    instance per 21-CONTEXT.md), so a full FS enumeration on every
        #    list is acceptable.
        base_dir = Path(settings.brand_kits_dir)
        fs_only_summaries: list[BrandKitSummary] = []
        if base_dir.exists():
            for child in sorted(base_dir.iterdir()):
                if not child.is_dir():
                    continue
                slug = child.name
                if not _SLUG_RE.match(slug):
                    continue
                if slug in all_db_slugs:
                    continue  # already in DB — skip (no double count)
                brand_json = child / "brand.json"
                if not brand_json.is_file():
                    continue
                try:
                    kit = BrandKit.model_validate_json(
                        brand_json.read_text(encoding="utf-8")
                    )
                except Exception:
                    # Corrupt brand.json — skip silently for list endpoint.
                    continue
                fs_only_summaries.append(
                    BrandKitSummary(
                        slug=slug,
                        name=kit.name,
                        source_url=kit.source_url,
                        scraped_at=kit.fetched_at,
                    )
                )
        fs_only_count = len(fs_only_summaries)

        # 3. Stable total BEFORE pagination.
        total = db_total + fs_only_count

        # 4. Paginate the merged list. We sort the DB-sourced page + FS-only
        #    tail together by scraped_at desc so recency ordering is global,
        #    not DB-first-then-FS-tail (IN-03 companion fix).
        #
        #    To keep the query cheap, we page the DB side natively (LIMIT +
        #    OFFSET against the indexed scraped_at column) and interleave
        #    FS-only summaries into the global sorted merge only for the
        #    slice that falls on the current page.
        db_rows = (
            (
                await session.execute(
                    select(BrandKitRecord)
                    .order_by(BrandKitRecord.scraped_at.desc())
                )
            )
            .scalars()
            .all()
        )
        db_summaries: list[BrandKitSummary] = [
            BrandKitSummary(
                slug=r.slug,
                name=r.name,
                source_url=r.source_url,
                scraped_at=r.scraped_at,
            )
            for r in db_rows
        ]

        # Merge + sort desc by scraped_at, then slice for the current page.
        merged = db_summaries + fs_only_summaries
        merged.sort(key=lambda s: s.scraped_at, reverse=True)
        page = merged[offset : offset + limit]

        return PaginatedBrandKits(
            items=page,
            total=total,
            limit=limit,
            offset=offset,
        )
    ```
