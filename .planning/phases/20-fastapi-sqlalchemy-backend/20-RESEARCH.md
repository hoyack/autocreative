# Phase 20: FastAPI + SQLAlchemy Backend — Research

**Researched:** 2026-04-22
**Domain:** Async Python HTTP API + relational persistence + background job queue
**Confidence:** HIGH (all core libraries verified on PyPI today; all patterns cross-checked against Context7-hosted official docs)

## Summary

Phase 20 wraps five already-working Python generators (`FlyerGenerator.generate`, `render_schema_brochure` + `generate_template_images`, `fetch_brand_kit`, `generate_post`, `generate_campaign`) with (a) a FastAPI async HTTP surface, (b) a SQLAlchemy 2.x async ORM with Alembic, (c) an arq + Redis job queue for long-running (60–300 s) ComfyCloud calls, and (d) a filesystem-backed artifact streaming endpoint. Every architectural decision is already locked by `20-CONTEXT.md`; the job of this research is to nail down the idioms and version pins so the planner can emit concrete tasks.

All five generators are already `async`, already use `httpx.AsyncClient`, and already use `structlog`. The FastAPI layer can therefore `await` them directly from worker tasks without a sync bridge. The Pydantic v2 request-body models already exist (`EventInput`, `PostBrief`, `BrochureContent`, etc.) and become FastAPI request schemas verbatim — no reshape is needed. The error hierarchy is already the right shape for a single exception-handler bank.

**Primary recommendation:** Adopt the FastAPI `lifespan` context manager pattern (deprecated: `@app.on_event`), use SQLAlchemy 2.x `async_sessionmaker` with a `get_session` dependency that yields `AsyncSession`, drive Alembic via the async `env.py` template (generated by `alembic init -t async alembic`), run arq's `WorkerSettings` as a **separate process** (share the same `async_sessionmaker` factory via a module-level singleton resolved from `AppSettings` at both `lifespan` startup and arq `on_startup`), and use `asgi-correlation-id` 4.3.4 for the `trace_id` middleware (one shared library, proven structlog integration) rather than hand-rolling it.

## Architectural Responsibility Map

| Capability | Primary Tier | Secondary Tier | Rationale |
|------------|-------------|----------------|-----------|
| HTTP request parsing + validation | API (FastAPI route) | — | Pydantic v2 at the route boundary is the canonical FastAPI pattern. Existing `EventInput` / `PostBrief` / `BrochureContent` are already Pydantic v2 — reuse as request bodies. |
| Error → HTTP status mapping | API (exception handlers) | — | Single bank of `@app.exception_handler` callbacks. Routes raise domain errors (`BrandKitError`, `ComfyError`); handlers translate to HTTP. |
| Long-running generator execution | Worker (arq) | — | ComfyCloud calls 60–300 s. Cannot run inside request lifetime. arq task functions wrap existing `async def` generators directly. |
| Job state transitions | Worker (arq) writing via SQLAlchemy | API (read-only poll) | Worker owns the write side of `JobRecord`; `GET /jobs/{id}` reads the current row. |
| Artifact storage | Filesystem (existing) | DB (metadata pointer) | `.brand-kits/<slug>/`, `.social-campaigns/<slug>/<campaign_id>/`, and `FLYER_OUTPUT_ROOT` remain canonical. `RenderRecord.file_path` is a pointer. |
| Artifact serving | API (FileResponse) | — | `GET /renders/{id}/image` resolves DB row, validates containment, streams from disk. |
| Request correlation | API middleware | structlog (already present) | `asgi-correlation-id` middleware + contextvar processor binds `trace_id` across FastAPI + structlog + existing generator logs without changing the generators. |
| Configuration | `AppSettings` (pydantic-settings) | — | Extends existing `Settings`; same `FLYER_` prefix; read once at import/startup. |
| Database migrations | Alembic (async env.py) | — | `alembic upgrade head` CLI at startup and in tests. |

### Alternatives Considered

| Instead of | Could Use | Tradeoff |
|------------|-----------|----------|
| asgi-correlation-id | Hand-rolled `BaseHTTPMiddleware` + structlog `bind_contextvars` | ~60 LOC saved; one less maintenance surface. Downside: one more dependency — but it is tiny (~200 LOC) and single-purpose. |
| asgi-correlation-id | `starlette-context` (0.5.1) | `starlette-context` is a general-purpose context plugin; overkill for a single trace_id. `asgi-correlation-id` does one thing. |
| arq | Celery | Celery is sync-first, needs a worker process AND a result backend AND a broker AND gevent/eventlet to bridge async — contradicts the all-async stack. |
| arq | FastAPI `BackgroundTasks` | BackgroundTasks runs in the request process — dies if uvicorn restarts mid-generation. ComfyCloud is 60–300 s; unsafe. [CITED: FastAPI docs — "BackgroundTasks is not a replacement for Celery/arq for long work"] |
| `FileResponse` | `StreamingResponse` | `FileResponse` auto-handles `Content-Length`, `Last-Modified`, `ETag`, and chunked streaming. `StreamingResponse` is for generator-produced payloads. PNG/PDF on disk → `FileResponse`. [CITED: fastapi.tiangolo.com/advanced/custom-response/#fileresponse] |
| SQLAlchemy 2.x async | Tortoise ORM / SQLModel | SQLModel reuses SQLA 2.x but hides migrations; Tortoise has a separate migration story (aerich) and fewer users. SQLA 2.x is the default a Python backend dev will know. |
| Alembic async template | SQLAlchemy `metadata.create_all` only | `create_all` is fine for tests but forbids column additions in prod without dropping. Alembic is the grown-up answer for the v1 SQLite → v2 Postgres story. |
| honcho / foreman for two-process dev | Makefile with `&` | `honcho` (PyPI, v2.0.0, Oct 2024) gives unified colored log streaming. Alternative: plain shell — adequate but uglier. `uv run` can launch honcho: `uv run honcho start -f Procfile`. Honcho is a 10-year-old stable package. [VERIFIED: PyPI `honcho==2.0.0`] |

**Flow — a `/api/v1/flyers` POST:**
1. CorrelationIdMiddleware stamps a ULID `X-Request-ID` header (or echoes the incoming one) and binds it to the contextvar — structlog processor surfaces it as `trace_id` in every log line.
2. Route handler validates `{event: EventInput, preset, brand_kit_slug?, accent?, max_bg_attempts?}` via Pydantic.
3. Route opens DB session, INSERTs `JobRecord(id=ULID, kind="flyer", status="queued", input_payload=JSON, trace_id=...)`, commits.
4. Route calls `arq_pool.enqueue_job("task_generate_flyer", job_id=..., ...)`.
5. Returns `202 {job_id}` immediately.
6. arq worker picks up, calls `task_generate_flyer(ctx, job_id, ...)`:
   - Opens its own `AsyncSession` from the worker-side sessionmaker,
   - UPDATEs `JobRecord` → `running`, commits,
   - Instantiates `FlyerGenerator(settings, http_client=ctx['http_client'])`,
   - `await generator.generate(event)` → `FlyerOutput`,
   - Writes PNG to `FLYER_OUTPUT_ROOT/<job_id>.png` on disk,
   - INSERTs `FlyerRecord` + `RenderRecord(file_path, vision_verdict=JSON, comfy_job_id)`,
   - UPDATEs `JobRecord` → `succeeded`, `completed_at`, `result_ref=<render_id>`.
7. Client polls `GET /jobs/{id}` until `status == "succeeded"`, reads `result_ref`, fetches `GET /renders/{render_id}/image`.

### Pattern 1: FastAPI lifespan with engine + arq pool

```python
# flyer_generator/api/lifespan.py
from contextlib import asynccontextmanager
from fastapi import FastAPI
from arq import create_pool
from arq.connections import RedisSettings
from flyer_generator.api.config import AppSettings
from flyer_generator.api.db import build_engine, build_sessionmaker

@asynccontextmanager
async def lifespan(app: FastAPI):
    settings = AppSettings()
    engine = build_engine(settings)
    app.state.sessionmaker = build_sessionmaker(engine)
    app.state.arq_pool = await create_pool(
        RedisSettings.from_dsn(str(settings.redis_url))
    )
    try:
        yield
    finally:
        await app.state.arq_pool.aclose()
        await engine.dispose()
```

Source: FastAPI official docs — "Lifespan Events" (https://fastapi.tiangolo.com/advanced/events/). The deprecated `@app.on_event("startup")` / `@app.on_event("shutdown")` pair must NOT be used — it is shown crossed-out in current Context7-fetched docs. [CITED: fastapi.tiangolo.com/advanced/events/]

### Pattern 2: `get_session` dependency

```python
# flyer_generator/api/db.py
from typing import AsyncIterator
from fastapi import Depends, Request
from sqlalchemy.ext.asyncio import (
    AsyncEngine, AsyncSession, async_sessionmaker, create_async_engine,
)
from flyer_generator.api.config import AppSettings

def build_engine(settings: AppSettings) -> AsyncEngine:
    # SQLite wants NullPool when shared across async tasks;
    # Postgres uses the default QueuePool sizing.
    kw = {"echo": False, "future": True}
    if settings.database_url.startswith("sqlite"):
        from sqlalchemy.pool import NullPool
        kw["poolclass"] = NullPool
        # required for multi-coroutine aiosqlite access:
        kw["connect_args"] = {"check_same_thread": False}
    return create_async_engine(settings.database_url, **kw)

def build_sessionmaker(engine: AsyncEngine) -> async_sessionmaker[AsyncSession]:
    return async_sessionmaker(engine, expire_on_commit=False, class_=AsyncSession)

async def get_session(request: Request) -> AsyncIterator[AsyncSession]:
    sessionmaker: async_sessionmaker[AsyncSession] = request.app.state.sessionmaker
    async with sessionmaker() as session:
        try:
            yield session
        except Exception:
            await session.rollback()
            raise
        else:
            await session.commit()
```

Source: SQLAlchemy 2.0 async docs + FastAPI dependency-yield docs. `expire_on_commit=False` is critical for async because objects would be lazily reloaded after commit, which would trigger I/O inside an already-closed session. [CITED: docs.sqlalchemy.org/en/20/orm/extensions/asyncio.html, fastapi.tiangolo.com/tutorial/dependencies/dependencies-with-yield/]

### Pattern 3: arq WorkerSettings sharing the sessionmaker

arq workers are a **separate process**. They cannot reach `app.state`. The sessionmaker must be rebuilt in the worker's `on_startup` from the same `AppSettings`:

```python
# flyer_generator/api/worker.py
from arq.connections import RedisSettings
import httpx
from flyer_generator.api.config import AppSettings
from flyer_generator.api.db import build_engine, build_sessionmaker
from flyer_generator.api.tasks import ALL_TASKS  # list of async functions

async def on_startup(ctx):
    settings = AppSettings()
    ctx["settings"] = settings
    ctx["engine"] = build_engine(settings)
    ctx["sessionmaker"] = build_sessionmaker(ctx["engine"])
    ctx["http_client"] = httpx.AsyncClient(follow_redirects=True, timeout=300)

async def on_shutdown(ctx):
    await ctx["http_client"].aclose()
    await ctx["engine"].dispose()

class WorkerSettings:
    functions = ALL_TASKS
    on_startup = on_startup
    on_shutdown = on_shutdown
    redis_settings = RedisSettings.from_dsn(str(AppSettings().redis_url))
    max_jobs = 4                  # ComfyCloud is the real concurrency bottleneck
    job_timeout = 600             # 10 min — covers a deep ComfyCloud queue
    keep_result = 3600
    max_tries = 1                 # we rely on llm_retry internals; don't double-retry
```

Source: arq official WorkerSettings examples (Context7: `/python-arq/arq`). **`max_tries = 1`** is a deliberate choice — the `flyer_generator/stages/llm_retry.py` primitive already implements retry-with-fallback-chain per model; re-running the whole task would double-charge ComfyCloud and thrash state transitions. [CITED: python-arq/arq docs]

### Pattern 4: Arq task → JobRecord state machine

```python
# flyer_generator/api/tasks/flyer.py
from datetime import datetime, timezone
from sqlalchemy import update
from flyer_generator.api.models import JobRecord, FlyerRecord, RenderRecord, JobStatus
from flyer_generator import FlyerGenerator, EventInput
from flyer_generator.errors import FlyerGeneratorError

async def task_generate_flyer(ctx, *, job_id: str, payload: dict) -> str:
    sessionmaker = ctx["sessionmaker"]
    settings = ctx["settings"]
    http_client = ctx["http_client"]

    # 1. running
    async with sessionmaker() as s:
        await s.execute(
            update(JobRecord)
            .where(JobRecord.id == job_id)
            .values(status=JobStatus.RUNNING, started_at=datetime.now(timezone.utc))
        )
        await s.commit()

    try:
        event = EventInput.model_validate(payload["event"])
        gen = FlyerGenerator(settings, http_client=http_client)
        out = await gen.generate(event)

        # 2. persist artifacts
        artifact_path = settings.artifact_root_flyer / f"{job_id}.png"
        artifact_path.parent.mkdir(parents=True, exist_ok=True)
        out.save(artifact_path)

        async with sessionmaker() as s:
            flyer = FlyerRecord(... )
            render = RenderRecord(
                kind="flyer_final",
                file_path=str(artifact_path),
                vision_verdict=out.final_vision_verdict.model_dump(mode="json"),
                comfy_job_id=out.final_vision_verdict.raw_response[:60],  # or carry from pipeline
            )
            s.add_all([flyer, render])
            await s.flush()           # assigns ids
            await s.execute(
                update(JobRecord)
                .where(JobRecord.id == job_id)
                .values(
                    status=JobStatus.SUCCEEDED,
                    completed_at=datetime.now(timezone.utc),
                    result_ref=str(render.id),
                )
            )
            await s.commit()
            return str(render.id)

    except FlyerGeneratorError as err:
        async with sessionmaker() as s:
            await s.execute(
                update(JobRecord)
                .where(JobRecord.id == job_id)
                .values(
                    status=JobStatus.FAILED,
                    completed_at=datetime.now(timezone.utc),
                    error_detail={"type": type(err).__name__, "message": str(err)},
                )
            )
            await s.commit()
        raise
```

**Key discipline:** the `queued → running` and `running → {succeeded, failed}` transitions each happen in their own `async with sessionmaker()` block. This guarantees every hop is committed even if the worker process is SIGTERM'd mid-generation — the client polling `/jobs/{id}` will see the correct state.

### Pattern 5: Exception-handler bank

```python
# flyer_generator/api/errors.py
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
from asgi_correlation_id import correlation_id
from flyer_generator.errors import (
    BrandKitError, FlyerGeneratorError, SocialError,
    BrandVoiceViolationError, ComfyError, LLMAPIError, LLMRateLimitError,
)

def _payload(exc: Exception, status: int) -> dict:
    return {
        "detail": str(exc),
        "error_type": type(exc).__name__,
        "trace_id": correlation_id.get() or "",
    }

def register_exception_handlers(app: FastAPI) -> None:
    @app.exception_handler(LLMRateLimitError)
    async def _rate_limit(request: Request, exc: LLMRateLimitError):
        headers = {"Retry-After": str(int(exc.retry_after_seconds or 1))}
        return JSONResponse(_payload(exc, 503), status_code=503, headers=headers)

    @app.exception_handler(LLMAPIError)          # catches all LLM sub-errors + VisionAPIError alias
    async def _llm_api(request: Request, exc: LLMAPIError):
        return JSONResponse(_payload(exc, 502), status_code=502)

    @app.exception_handler(ComfyError)           # base covers submit/timeout/download/job-failed
    async def _comfy(request: Request, exc: ComfyError):
        return JSONResponse(_payload(exc, 502), status_code=502)

    @app.exception_handler(BrandVoiceViolationError)
    async def _voice(request: Request, exc: BrandVoiceViolationError):
        return JSONResponse(_payload(exc, 422), status_code=422)

    # Mapping note: BrandKitNotFoundError does NOT exist in the hierarchy today.
    # See "Open Questions" — either add it or route `BrandKitError` to 404 for
    # the "load by slug" path only (404) vs 400 for scrape failures. See below.

    @app.exception_handler(BrandKitError)
    async def _brand_kit(request: Request, exc: BrandKitError):
        # Default to 400; routes that perform lookup may re-raise as HTTPException(404).
        return JSONResponse(_payload(exc, 400), status_code=400)

    @app.exception_handler(SocialError)          # covers PostValidationError, PlatformUnsupportedError, etc.
    async def _social(request: Request, exc: SocialError):
        return JSONResponse(_payload(exc, 400), status_code=400)

    @app.exception_handler(FlyerGeneratorError)  # catch-all for the rest of the tree
    async def _flyer_base(request: Request, exc: FlyerGeneratorError):
        return JSONResponse(_payload(exc, 400), status_code=400)
```

**Ordering matters:** more-specific classes first (FastAPI matches the *first* registered handler whose `isinstance(exc, X)` is true). `LLMRateLimitError` before `LLMAPIError`, `BrandVoiceViolationError` before `BrandKitError`, all before `FlyerGeneratorError`. [CITED: FastAPI handling-errors docs]

### Pattern 6: Safe artifact serving

```python
# flyer_generator/api/routes/renders.py
from pathlib import Path
from fastapi import APIRouter, Depends, HTTPException
from fastapi.responses import FileResponse
from sqlalchemy.ext.asyncio import AsyncSession
from flyer_generator.api.db import get_session
from flyer_generator.api.config import AppSettings, get_settings
from flyer_generator.api.models import RenderRecord

router = APIRouter()

ALLOWED_EXT_MIME = {".png": "image/png", ".pdf": "application/pdf", ".jpg": "image/jpeg"}

def _is_within(path: Path, root: Path) -> bool:
    try:
        path.resolve(strict=True).relative_to(root.resolve(strict=True))
        return True
    except (ValueError, FileNotFoundError):
        return False

@router.get("/renders/{render_id}/image")
async def get_render_image(
    render_id: str,
    session: AsyncSession = Depends(get_session),
    settings: AppSettings = Depends(get_settings),
) -> FileResponse:
    record = await session.get(RenderRecord, render_id)
    if record is None:
        raise HTTPException(status_code=404, detail="render not found")
    candidate = Path(record.file_path)
    if not any(
        _is_within(candidate, Path(root))
        for root in (
            settings.artifact_root_flyer,
            settings.artifact_root_brochure,
            settings.brand_kits_dir,
            settings.social_campaigns_dir,
        )
    ):
        # treat "outside allowed roots" as 404 (don't leak filesystem shape)
        raise HTTPException(status_code=404, detail="render not found")
    media_type = ALLOWED_EXT_MIME.get(candidate.suffix.lower(), "application/octet-stream")
    return FileResponse(
        path=candidate,
        media_type=media_type,
        headers={"Content-Disposition": f'inline; filename="{candidate.name}"'},
    )
```

Source: FastAPI FileResponse docs (https://fastapi.tiangolo.com/advanced/custom-response/#fileresponse). `Path.resolve(strict=True)` then `.relative_to()` is the canonical industry pattern for defending against `..` and symlink traversal. [CITED: fastapi.tiangolo.com/advanced/custom-response/; OWASP "Path Traversal" cheat sheet]

### Pattern 7: Alembic async env.py

Generate via `uv run alembic init -t async alembic` — the `async` template has existed since Alembic 1.8.0 and is explicitly designed for this case. Then edit `env.py` to import your `Base` metadata:

```python
# alembic/env.py (scaffold from `alembic init -t async alembic`, then edit)
from logging.config import fileConfig
from sqlalchemy.ext.asyncio import async_engine_from_config
from sqlalchemy import pool
from alembic import context

from flyer_generator.api.config import AppSettings
from flyer_generator.api.models import Base  # re-exports DeclarativeBase + all models

config = context.config
settings = AppSettings()
config.set_main_option("sqlalchemy.url", settings.database_url)

if config.config_file_name is not None:
    fileConfig(config.config_file_name)

target_metadata = Base.metadata

def run_migrations_offline() -> None:
    context.configure(
        url=settings.database_url,
        target_metadata=target_metadata,
        literal_binds=True,
        dialect_opts={"paramstyle": "named"},
    )
    with context.begin_transaction():
        context.run_migrations()

def do_run_migrations(connection) -> None:
    context.configure(connection=connection, target_metadata=target_metadata)
    with context.begin_transaction():
        context.run_migrations()

async def run_migrations_online() -> None:
    connectable = async_engine_from_config(
        config.get_section(config.config_ini_section, {}),
        prefix="sqlalchemy.",
        poolclass=pool.NullPool,
    )
    async with connectable.connect() as connection:
        await connection.run_sync(do_run_migrations)
    await connectable.dispose()

import asyncio
if context.is_offline_mode():
    run_migrations_offline()
else:
    asyncio.run(run_migrations_online())
```

Source: Alembic changelog explicitly documents the async template (`alembic init -t async`) and the cookbook entry "Using Asyncio with Alembic" (https://alembic.sqlalchemy.org/en/latest/cookbook.html#using-asyncio-with-alembic). `pool.NullPool` is required — Alembic runs a one-off connection, pooling is pointless and would leak. [CITED: alembic.sqlalchemy.org/en/latest/cookbook.html#using-asyncio-with-alembic, SQLAlchemy changelog 1.11.0]

### Pattern 8: Request correlation + structlog wiring

Use `asgi-correlation-id` with a ~5-line structlog processor:

```python
# flyer_generator/api/middleware.py
from asgi_correlation_id import CorrelationIdMiddleware, correlation_id
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from flyer_generator.api.config import AppSettings

def install_middleware(app: FastAPI, settings: AppSettings) -> None:
    # CorrelationIdMiddleware generates a UUID if none is present and
    # echoes the incoming X-Request-ID header. It also populates a
    # contextvar named `correlation_id` that structlog can read.
    app.add_middleware(CorrelationIdMiddleware, header_name="X-Request-ID")
    app.add_middleware(
        CORSMiddleware,
        allow_origins=settings.cors_origins,
        allow_credentials=True,
        allow_methods=["*"],
        allow_headers=["*"],
    )

# flyer_generator/logging_config.py (ADD to existing processor chain)
def _add_correlation(logger, method_name, event_dict):
    if cid := correlation_id.get():
        event_dict.setdefault("trace_id", cid)
    return event_dict

# inject _add_correlation before existing processors list
```

**Compatibility with existing generator code:** The generator modules already call `structlog.get_logger()` inside their own flow. They don't bind `trace_id` themselves — the request's `trace_id` flows through the contextvar, so every log line the generator emits while servicing a request will be stamped with the correct `trace_id` without modifying the generators. Verified by reading `flyer_generator/logging_config.py` — `structlog.contextvars.merge_contextvars` is already first in the processor list, so `correlation_id`'s contextvar is picked up for free. [VERIFIED: flyer_generator/logging_config.py:14]

### Anti-Patterns to Avoid

- **`@app.on_event("startup")` / `@app.on_event("shutdown")`.** Deprecated since FastAPI 0.93; migration path is `lifespan`. [CITED: fastapi.tiangolo.com/advanced/events/]
- **Using `AsyncSession` as a long-lived app-global.** Sessions are per-request; the sessionmaker is the app-global.
- **`session.commit()` inside a generator that also opens and re-opens sessions.** The `get_session` dep already commits on non-exception exit; routes should NOT manually `await session.commit()` again unless they have a reason.
- **Sharing the `ArqRedis` pool with the worker.** The worker has its own Redis connection from its own `RedisSettings`. Only the FastAPI side uses the `app.state.arq_pool`.
- **Sync SQLAlchemy call inside an async route.** Even `session.query(...)` (without `await`) is a footgun — on aiosqlite it blocks the event loop. Use `await session.execute(select(...))`.
- **Putting PNG bytes in the DB `JSON` column.** `JobRecord.input_payload` stores the request body; PNG bytes always go to disk and a `RenderRecord.file_path` pointer. Concretely: do NOT serialize `FlyerOutput.png_bytes` to DB — `.save(path)` to disk, store the path only.
- **Running `BackgroundTasks` for ComfyCloud.** Any task longer than ~10 s must go to arq.
- **`pool_pre_ping=True` on aiosqlite.** Overhead; SQLite has no real connection to ping. Use only on Postgres.
- **Returning `StreamingResponse` for files on disk.** `FileResponse` handles `Last-Modified` / `ETag` / range requests for free. [CITED: fastapi.tiangolo.com/advanced/custom-response/]
- **Missing `expire_on_commit=False` on the sessionmaker.** Without it, every `ORM.attribute` access after `commit()` triggers a lazy re-SELECT, which in async land throws `MissingGreenlet`.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Request-ID / trace_id middleware | Custom `BaseHTTPMiddleware` + contextvar | `asgi-correlation-id` (v4.3.4) | Maintained, includes the structlog processor we need out of the box, handles header-name config, UUID generation, and inbound echo correctly. [VERIFIED: PyPI] |
| Async job queue | Custom asyncio task gatherer | `arq` (v0.28.0) | You need Redis-backed persistence across process restarts; arq gives it in ~200 LOC of config. Rolling this is a classic "you ended up writing Celery" mistake. |
| Schema migrations | Ad-hoc SQL in a startup script / `metadata.create_all` in prod | `alembic` (v1.18.4) async template | Alembic's async template is a 2-command scaffold. Hand-rolling loses: autogenerate diff, revision DAG, downgrade paths. |
| Per-request DB session management | Manual `with session` in every route | `get_session` FastAPI dependency with yield | FastAPI's dep-injection already gives: per-request lifetime, commit-on-success, rollback-on-exception, automatic cleanup. |
| OpenAPI schema | Hand-maintained JSON / YAML | FastAPI's auto-generation from Pydantic models | Phase 21's client generator depends on accurate OpenAPI; FastAPI emits it for free. |
| Pagination | Custom skip/limit + total counting | `fastapi-pagination` (optional) or 20-line helper | For `GET /brand-kits` only; out of scope to pick in research — planner's discretion per CONTEXT.md. |
| ULID generation | Custom base32 encoder | `python-ulid` (already a dep) | Already in pyproject. `JobRecord.id` defaults to `ulid.new().str`. |
| Two-process dev launcher | Bash `&` with no log coloring | `honcho` (+ `Procfile`) | One `uv run honcho start` command, colored prefixed logs per process, Ctrl-C kills both. |
| Path-traversal defense | Regex-based "does it contain `..`" | `Path.resolve(strict=True).relative_to(root.resolve())` | Regex misses symlinks; `resolve(strict=True)` follows and then verifies containment against an actual root. |
| Error response format | Per-route try/except | Centralized `@app.exception_handler` bank | One bank, one shape, one test per mapping. Avoids the 20+ try/except blocks that would otherwise litter the routes. |

**Key insight:** five of the "don't hand-roll" items (arq, alembic, asgi-correlation-id, python-ulid, honcho) are each 1 line of pyproject.toml and 5–20 lines of wiring. Trying to replace any of them with in-repo code trades a known-good dependency for a net-new subtle-bug surface. The cost-benefit is overwhelmingly on the library side.

## Common Pitfalls

### Pitfall 1: Shared engine across uvicorn worker processes
**What goes wrong:** uvicorn's `--workers N` spawns N processes. A module-level `engine = create_async_engine(...)` would be forked (on Linux) and each child would inherit a broken connection state — SQLite silently corrupts, Postgres throws "connection does not exist".
**Why it happens:** Forked connections share TCP sockets.
**How to avoid:** Build the engine inside `lifespan` (per-process), never at module top level. `app.state.engine` is fine because each worker has its own `app.state`.
**Warning signs:** Random "database is locked" (SQLite) or `InterfaceError: cannot perform operation: another operation is in progress` (asyncpg).

### Pitfall 2: aiosqlite `check_same_thread` default
**What goes wrong:** aiosqlite dispatches DB work to a background thread. Default `check_same_thread=True` rejects it.
**How to avoid:** `connect_args={"check_same_thread": False}` in `create_async_engine` for SQLite URLs. Don't set for Postgres.
**Warning signs:** `ProgrammingError: SQLite objects created in a thread can only be used in that same thread`.

### Pitfall 3: `expire_on_commit=True` triggers lazy loads after commit
**What goes wrong:** The default (`True`) marks every attribute as expired after commit. Reading `job.id` after `await session.commit()` triggers a fresh SELECT — async-unsafe — and throws `MissingGreenlet: greenlet_spawn has not been called`.
**How to avoid:** `async_sessionmaker(engine, expire_on_commit=False)`. **Non-optional for async SQLAlchemy.** [CITED: SQLAlchemy 2.0 async docs — "ORM Querying Guide"]
**Warning signs:** `MissingGreenlet` errors; explicit `await session.refresh(obj)` calls proliferating as a workaround.

### Pitfall 4: arq task retried by default on exception
**What goes wrong:** `max_tries=5` (arq default until 0.26) retries the whole task body. But our pipeline already charges ComfyCloud per submission — a retry doubles the cost.
**How to avoid:** `max_tries=1` in WorkerSettings. `llm_retry.py` handles retryable failures. If the task raises, it's a "real" failure and should fail the job.
**Warning signs:** ComfyCloud bill spikes; multiple ComfyCloud jobs for one API call.

### Pitfall 5: Worker can't access `app.state`
**What goes wrong:** Someone writes `ctx["sessionmaker"] = app.state.sessionmaker` inside an arq task, forgetting that the worker is a separate OS process.
**How to avoid:** Build the sessionmaker in `on_startup(ctx)` from `AppSettings()` — NOT from the FastAPI app.
**Warning signs:** `NameError: name 'app' is not defined` at worker startup, or worse — the import works but every DB call fails because the engine is None.

### Pitfall 6: Testing with a real Redis
**What goes wrong:** CI has no Redis; tests hang or fail with connection-refused.
**How to avoid:** Three options in descending simplicity: (a) mock `arq.connections.ArqRedis.enqueue_job` with a stub that directly awaits the task function — validates end-to-end logic but bypasses Redis; (b) use `fakeredis` library to substitute the Redis connection; (c) spin up Redis in GitHub Actions via `services: redis: {image: redis:7}`. **Recommended for Phase 20: (a)** — a `@pytest.fixture` that monkeypatches `enqueue_job` to run the task inline with a fresh DB session. This is what the existing `tests/social/test_integration.py` pattern does for ComfyCloud (mocks at the client boundary, not the network).
**Warning signs:** API tests hang for 30 s before timing out.

### Pitfall 7: CORS credentials + wildcard incompatible
**What goes wrong:** `allow_origins=["*"]` + `allow_credentials=True` is a spec violation — browsers reject it.
**How to avoid:** Read concrete origins from `FLYER_CORS_ORIGINS` (default `["http://localhost:5173"]` for Vite). Don't use wildcards when credentials are on.

### Pitfall 8: Path traversal via `..` survives naive `os.path.join`
**What goes wrong:** `Path(settings.artifact_root) / user_supplied_filename` — if `user_supplied_filename == "../../../etc/passwd"`, the resulting path is a traversal.
**How to avoid:** Never accept a filename from the client. The client supplies a `render_id`; the server resolves `file_path` from the DB. The only path resolution is `Path(record.file_path).resolve(strict=True).relative_to(allowed_root.resolve())` — and any failure is 404.
**Warning signs:** `FileResponse` is constructed with a path containing `..`.

### Pitfall 9: `BrandKitNotFoundError` doesn't actually exist
**What goes wrong:** CONTEXT.md and the error-mapping table both reference `BrandKitNotFoundError`, but `flyer_generator/errors.py` only defines `BrandKitError`, `BrandKitScrapeError`, `BrandKitContrastError`, `BrandKitAuditError`, `BrandVoiceViolationError`. A naïve implementation of the handler bank will fail with `NameError`.
**How to avoid:** Either (a) add a new `BrandKitNotFoundError(BrandKitError)` subclass in `flyer_generator/errors.py` and raise it from `load_brand_kit` when the slug doesn't resolve, or (b) handle the "not found" case as a route-level `HTTPException(404)` in `routes/brand_kits.py` without touching the error tree. **Recommendation:** (a) — matches the spec table and keeps the mapping symmetric. Listed in Open Questions.

### Pitfall 10: structlog's `add_log_level` already present — don't double-add
**What goes wrong:** Someone adds the `asgi-correlation-id` example verbatim (which includes `structlog.stdlib.add_log_level`), but the existing `logging_config.py` already adds `structlog.processors.add_log_level`. Result: every log line has two level fields or a KeyError.
**How to avoid:** Only add the `_add_correlation` processor to the existing chain; do NOT rewrite the whole processor list from the asgi-correlation-id README.
**Warning signs:** Tests assert on log output and fail with "duplicate key".

### Pitfall 11: SQLite Alembic "batch mode" vs Postgres "direct ALTER"
**What goes wrong:** A simple `op.alter_column(...)` works on Postgres but silently fails (or drops data) on SQLite.
**How to avoid:** Use `op.batch_alter_table(...)` in every migration that mutates existing columns. The initial migration is pure CREATE TABLE so it's fine; all _subsequent_ migrations need batch mode. Document this convention in `alembic/env.py` as a comment.

### Enqueuing a job from a route

```python
# flyer_generator/api/routes/flyers.py
from fastapi import APIRouter, Depends, Request, status
from sqlalchemy.ext.asyncio import AsyncSession
from flyer_generator.api.db import get_session
from flyer_generator.api.models import JobRecord, JobStatus, JobKind
from flyer_generator.api.schemas.flyers import FlyerCreateRequest, JobCreated
import ulid

router = APIRouter()

@router.post("/flyers", status_code=status.HTTP_202_ACCEPTED, response_model=JobCreated)
async def create_flyer(
    body: FlyerCreateRequest,
    request: Request,
    session: AsyncSession = Depends(get_session),
) -> JobCreated:
    job_id = str(ulid.new())
    session.add(JobRecord(
        id=job_id,
        kind=JobKind.FLYER,
        status=JobStatus.QUEUED,
        input_payload=body.model_dump(mode="json"),
    ))
    await session.flush()
    # get_session commits on successful exit; enqueue AFTER commit to avoid
    # a race where the worker picks up the job before the row is visible.
    await session.commit()

    await request.app.state.arq_pool.enqueue_job(
        "task_generate_flyer",
        job_id=job_id,
        payload=body.model_dump(mode="json"),
    )
    return JobCreated(job_id=job_id)
```

### JobRecord with enums and ULID

```python
# flyer_generator/api/models/job.py
import enum
from sqlalchemy import String, DateTime, JSON, Enum as SAEnum
from sqlalchemy.orm import Mapped, mapped_column
from flyer_generator.api.models.base import Base
from datetime import datetime

class JobKind(str, enum.Enum):
    BRAND_KIT = "brand_kit"
    FLYER = "flyer"
    BROCHURE = "brochure"
    SOCIAL_POST = "social_post"
    SOCIAL_CAMPAIGN = "social_campaign"

class JobStatus(str, enum.Enum):
    QUEUED = "queued"
    RUNNING = "running"
    SUCCEEDED = "succeeded"
    FAILED = "failed"
    CANCELLED = "cancelled"

class JobRecord(Base):
    __tablename__ = "jobs"

    id: Mapped[str] = mapped_column(String(26), primary_key=True)          # ULID
    kind: Mapped[JobKind] = mapped_column(SAEnum(JobKind), index=True)
    status: Mapped[JobStatus] = mapped_column(SAEnum(JobStatus), index=True, default=JobStatus.QUEUED)
    started_at: Mapped[datetime | None] = mapped_column(DateTime(timezone=True), nullable=True)
    completed_at: Mapped[datetime | None] = mapped_column(DateTime(timezone=True), nullable=True)
    error_detail: Mapped[dict | None] = mapped_column(JSON, nullable=True)
    result_ref: Mapped[str | None] = mapped_column(String(26), nullable=True)   # RenderRecord.id
    input_payload: Mapped[dict] = mapped_column(JSON, default=dict)
    trace_id: Mapped[str | None] = mapped_column(String(36), nullable=True, index=True)
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), default=lambda: datetime.utcnow())
```

Use Python `str, enum.Enum` (rather than bare `enum.Enum`) so `model_dump(mode="json")` emits string values, not `"JobStatus.RUNNING"`. [CITED: Pydantic v2 enum handling, pydantic docs]

## Open Questions

1. **`BrandKitNotFoundError` does not exist in the error hierarchy today.** The CONTEXT.md mapping table says `BrandKitNotFoundError → 404`, but `flyer_generator/errors.py` only defines `BrandKitError` and its scrape/contrast/audit/voice subclasses. Two options:
   - **Option A (recommended):** Add a new `BrandKitNotFoundError(BrandKitError)` class in `flyer_generator/errors.py`. Have `load_brand_kit(slug)` raise it when `resolve_kit_dir(slug)` fails to find `brand.json`. Existing callers that catch `BrandKitError` still match (since it's a subclass).
   - **Option B:** Skip the new class; handle the 404 in the route handler directly via `raise HTTPException(404, ...)` when `load_brand_kit` returns nothing. Less clean, but zero changes to the errors module.
   - **Recommendation:** Option A.
   - **Decision owner:** planner (tasked under "Claude's discretion" per CONTEXT.md).

2. **`render_schema_brochure` is a sync function** (signature from Grep: `def render_schema_brochure(template, content, *, images, textures, logo_bytes, accent_override)`). The worker's arq task is `async def`. Either (a) wrap it in `asyncio.to_thread(...)` inside the task (canonical; non-blocking for the event loop), or (b) make `render_schema_brochure` itself async. Rendering is pure CPU (SVG build + cairosvg rasterize) — it doesn't need to be async. **Recommendation: (a) `asyncio.to_thread`** in the brochure task, and leave `render_schema_brochure` sync. `generate_template_images` IS already async (it drives ComfyCloud), so that call stays direct.
   - **Decision owner:** planner.

3. **How does `GET /brand-kits` "fuse" existing on-disk kits with DB rows?** The spec text in CONTEXT.md `<specifics>` is terse: "should fuse DB rows with any `.brand-kits/*/` directories that predate the DB — gracefully import existing kits on first read." Two concrete implementations:
   - **Lazy:** On every `GET /brand-kits`, list `.brand-kits/*/`, for each slug not already in DB, synthesize a `BrandKitRecord` in-memory (no insert). Pros: safe, stateless. Cons: hides the fact that DB is not authoritative.
   - **Import-on-first-touch:** On startup or first read, scan `.brand-kits/*/`, insert `BrandKitRecord`s for missing slugs, commit. Pros: DB is authoritative after first run. Cons: requires a startup hook + conflict handling.
   - **Recommendation:** Lazy — simpler, fewer invariants to maintain. A "full import" CLI command can land in a later phase if needed.
   - **Decision owner:** planner.

4. **Pagination shape for `GET /brand-kits`.** `fastapi-pagination` is a battle-tested library, but adds a dependency for one route. Options: (a) ad-hoc `?limit=&offset=` with `{items, total}` response, (b) `fastapi-pagination`. CONTEXT.md explicitly lists this as planner's discretion.
   - **Recommendation:** (a). v1 has at most a dozen brand kits; a 20-line helper is fine.

5. **WebSocket `/jobs/{id}/stream`?** HANDOFF.md §7 mentioned it as optional. CONTEXT.md defers it. Recommend staying deferred — polling is adequate, and Phase 21's UI can add WS later without a breaking API change.

6. **`alembic upgrade head` on app startup, or separate command?** Running migrations in `lifespan` startup is convenient for dev (auto-migrate on reload) but dangerous in prod (races between workers). Convention: DO NOT auto-migrate in `lifespan`. Document "run `uv run alembic upgrade head` before `uv run uvicorn ...`" in the README.
   - **Recommendation:** Separate command. CI / Makefile target.

7. **Worker DB URL — same as app or separate?** Worker connects to the same DB as the app. Reuse `FLYER_DATABASE_URL`. The only reason to split would be read replicas (out of scope v1).

8. **Test scope — how many `tests/api/` files for the ≥50-test target?** CONTEXT.md lists 9 files (`test_app_smoke.py`, `test_error_mapping.py`, `test_brand_kits_routes.py`, `test_flyer_routes.py`, `test_brochure_routes.py`, `test_social_routes.py`, `test_jobs_routes.py`, `test_renders_routes.py`, `test_worker_tasks.py`). ~6 tests per file averages 54. Planner should confirm the split or allow consolidation.

9. **`docker-compose.yml` version pin.** CONTEXT.md says `postgres:16` and `redis:7`. Ensure these are the latest LTS as of Phase 20 execution — check https://hub.docker.com for `:16` and `:7` tags.

### Known Threat Patterns for FastAPI + SQLAlchemy + arq

| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| Path traversal on `GET /renders/{id}/image` | Tampering / Information Disclosure | `Path.resolve(strict=True).relative_to(allowed_root.resolve(strict=True))` with 404 on mismatch; never accept filenames from the client. [CITED: OWASP Path Traversal] |
| SQL injection | Tampering | SQLAlchemy ORM parameterizes all queries. Never use `.execute(text(...))` with f-string interpolation; use bind params. |
| SSRF on `POST /brand-kits/fetch` | Spoofing / Information Disclosure | Already mitigated by Phase 18 `fetch_brand_kit` — the scraper SSRF-gates the initial URL and every discovered logo URL (confirmed in `flyer_generator/brand_kit/scraper.py:280+`). The API route inherits this; no new mitigation needed. |
| DoS via long-running ComfyCloud call blocking event loop | DoS | Work offloaded to arq worker; request returns 202 immediately. Worker has `job_timeout=600`. |
| DoS via unbounded request body | DoS | FastAPI doesn't cap bodies by default. Add `uvicorn --limit-max-requests` + a Starlette `ContentSizeLimitMiddleware` if large bodies become a concern. For v1 bodies are tiny (~5 KB structured JSON); skip. |
| Unbounded CORS | Spoofing | `FLYER_CORS_ORIGINS` is a concrete list (default `["http://localhost:5173"]`); never `["*"]` with credentials. |
| Secrets leaked in error responses | Information Disclosure | `SecretStr` masks in `repr`; `{detail, error_type, trace_id}` format excludes `context` dict. Exception handlers only emit `str(exc)` (the message) — no traceback or `exc.context`. |
| Enumeration of `render_id` / `job_id` | Information Disclosure | Both are 26-char ULIDs — enumeration is computationally infeasible. |
| Redis command injection from API input | Tampering | arq only enqueues via typed parameters; we never pass user input as a Redis command. |
| Bleeding `FLYER_*_API_KEY` via OpenAPI schema | Information Disclosure | `AppSettings` inherits from `BaseSettings` — NOT a Pydantic model exposed in any route. `/docs` only surfaces route schemas. |

**Threat model gap (explicit, accepted per CONTEXT.md):** No auth layer. Any process on the local network can POST to any `/api/v1/*` endpoint. This is documented as an explicit v1 assumption; Phase 21+ will introduce auth.
